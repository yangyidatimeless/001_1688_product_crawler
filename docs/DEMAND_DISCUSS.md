## 🎯 需求背景

**业务目标**：开发一个 1688 商品爬虫工具，用于选品分析，可抓取销量好的商品数据。

**使用场景**：
- 长期监控 + 定期选品调研结合
- 初期：每周 1-2 次选品分析，抓取特定品类（如拖把、家居用品）的爆款数据
- 后期：可能对选定商品进行价格/销量趋势跟踪（二期规划）

## ✅ 核心共识与决策

### 1. 技术方案

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 爬虫框架 | Python 3.11 + Playwright | 异步并发，Stealth 模式绕过反爬 |
| 数据存储 | SQLite + JSONL 双存储 | SQLite 查询去重，JSONL 备份交换 |
| 反爬策略 | 代理池 + 频率控制 + UA 轮换 | 免费代理测试 2-3 天，成功率<70% 转商业 |
| 配置方式 | config.yaml + 命令行参数 | 支持 `--preview` 预览模式 |
| 日志系统 | logging + tqdm 进度条 | 独立日志文件，实时进度显示 |

### 2. 数据采集字段

**必填字段**：
- 商品标题、价格（min/max）、销量、供应商名称、供应商等级
- 商品 URL、主图 URL、缩略图 URL、采集时间戳

**选填字段**：
- 评价数量、复购率、发货地、最小起订量、商品标签/属性

### 3. 数据策略

- **更新策略**：保留历史记录 + 版本标记（同一商品多次抓取时新增记录而非覆盖）
- **去重逻辑**：单次任务内按商品 ID 去重，跨任务保留历史
- **数据库设计**：SQLite 表包含 `product_id` + `collected_at` 联合唯一键
- **标记字段**：增加 `is_latest` 标记，方便快速查询最新数据

### 5. GUI 路线规划

**一期（MVP）**：命令行工具
- 通过 `config.yaml` 配置关键词、抓取数量、频率等参数
- 命令行启动：`python crawler.py --keyword "拖把" --limit 100`
- 进度条显示：使用 `tqdm` 库，实时显示抓取进度

**二期（规划）**：Web 界面
- 技术栈：Vue 3 + Element Plus + FastAPI + ECharts
- 核心功能：任务配置、进度监控、数据浏览、导出下载、数据看板
- 部署：Docker 容器化，一键部署到服务器
- 负责人：少平（前端）+ 允灿（后端 API）

### 6. 预算批准

| 项目 | 预算 | 使用条件 |
|------|------|----------|
| 代理 IP 服务 | 200-300 元/月 | 免费代理测试 2-3 天，成功率<70% 时启用 |
| 推荐服务商 | 芝麻代理/快代理/讯代理 | 主流服务商，稳定性有保障 |

### 7. QA 测试计划

**测试介入时间点**：
- Day 3 晚：参与 MVP 演示评审，重点检查数据质量和字段完整性
- Day 4-5：配合允灿的单元测试，编写独立验证脚本
- Day 6：开始完整 QA 测试（功能测试 + 稳定性测试 + 异常场景测试）

**测试用例准备**：
- 测试关键词列表（10-20 个不同品类）
- 边界场景用例：销量为 0、价格区间值、无供应商信息、图片缺失等
- 异常场景用例：网络中断、页面结构变化、滑块验证触发等

**数据验证脚本**：
- 字段完整性校验（必填字段不能为空）
- 数据合理性校验（价格>0、销量>=0、URL 格式正确）
- 去重逻辑校验（同一任务内无重复商品 ID）
- 输出数据质量报告（成功率、缺失字段统计、异常数据标记）

## ✅ 架构决策记录 (ADR)

| 编号 | 决策点 | 最终选择 | 理由与权衡 |
| :--- | :--- | :--- | :--- |
| **ADR-01** | 爬虫框架选型 | **Python + Playwright** | 相比 Selenium 更轻量，Stealth 模式可绕过大部分反爬，支持异步并发 |
| **ADR-02** | 数据存储方案 | **SQLite + JSONL 双存储** | SQLite 便于查询去重，JSONL 便于数据交换和备份，兼顾两者优势 |
| **ADR-03** | 反爬策略 | **代理池 + 频率控制 + UA 轮换** | 多层防护，免费代理先测试，成功率<70% 转商业服务，控制成本 |
| **ADR-04** | 交互方式 | **命令行 MVP → Web 界面二期** | 快速验证核心价值，避免前期过度投入 UI，二期由少平负责前端 |
| **ADR-05** | 数据更新策略 | **保留历史 + is_latest 标记** | 支持价格/销量趋势分析，同时方便查询最新数据 |
| **ADR-06** | 开发语言 | **Python 3.11** | 爬虫生态成熟，Playwright 支持好，团队熟悉 |
