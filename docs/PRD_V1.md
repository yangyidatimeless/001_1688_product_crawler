## 🎯 产品概述

### 1.1 业务目标

开发一个 1688 商品爬虫工具，用于选品分析，可抓取销量好的商品数据，支持长期监控和定期选品调研。

### 1.2 使用场景

- **初期**：每周 1-2 次选品分析，抓取特定品类（如拖把、家居用品）的爆款数据
- **后期**：对选定商品进行价格/销量趋势跟踪（二期规划）

### 1.3 目标用户

- 电商选品运营人员
- 需要分析 1688 商品数据的业务人员

## 📦 核心功能 (MVP)

### US_01: 基础商品爬取

**作为** 选品运营人员  
**我希望** 输入关键词后能自动抓取 1688 商品列表  
**以便** 快速获取目标品类的商品数据

**验收标准 (DoD)**：
- [ ] 支持命令行输入关键词（如 `--keyword "拖把"`）
- [ ] 支持指定抓取数量（如 `--limit 50`）
- [ ] 成功抓取商品标题、价格、销量、供应商名称、供应商等级
- [ ] 成功抓取商品 URL、主图 URL、缩略图 URL
- [ ] 自动记录采集时间戳
- [ ] 数据保存到 SQLite 数据库和 JSONL 文件
- [ ] 命令行显示实时进度条（tqdm）

---

### US_02: 反爬策略实现

**作为** 系统使用者  
**我希望** 爬虫能自动绕过 1688 的反爬机制  
**以便** 稳定持续地抓取数据

**验收标准 (DoD)**：
- [ ] 实现 UA 轮换（至少 5 个不同 User-Agent）
- [ ] 实现请求频率控制（可配置，默认 2-5 秒/请求）
- [ ] 集成代理 IP 支持（支持免费代理池）
- [ ] 使用 Playwright Stealth 模式
- [ ] 遇到滑块验证时能识别并记录日志
- [ ] 代理失败时自动切换下一个

---

### US_03: 数据存储与去重

**作为** 数据分析师  
**我希望** 抓取的数据能自动去重并保留历史版本  
**以便** 后续进行趋势分析且避免重复数据

**验收标准 (DoD)**：
- [ ] SQLite 数据库包含 `product_id` + `collected_at` 联合唯一键
- [ ] 单次任务内按商品 ID 去重，不重复抓取
- [ ] 跨任务保留历史记录，不覆盖旧数据
- [ ] 每条记录包含 `is_latest` 标记字段
- [ ] 同步写入 JSONL 文件作为备份
- [ ] 支持查询最新数据（`WHERE is_latest = 1`）

---

### US_04: 配置管理

**作为** 系统使用者  
**我希望** 通过配置文件和命令行参数自定义爬取行为  
**以便** 灵活适配不同场景

**验收标准 (DoD)**：
- [ ] 支持 `config.yaml` 配置文件
- [ ] 配置文件包含：关键词列表、抓取数量、频率、代理设置
- [ ] 命令行参数可覆盖配置文件（如 `--preview` 预览模式）
- [ ] 预览模式：只抓取前 5 条数据用于测试
- [ ] 配置文件不存在时使用默认值

---

### US_05: 日志与异常处理

**作为** 开发/运维人员  
**我希望** 系统有完善的日志记录和异常处理  
**以便** 快速定位问题和监控运行状态

**验收标准 (DoD)**：
- [ ] 独立日志文件（按日期命名，如 `crawler_2026-04-01.log`）
- [ ] 日志包含：INFO（正常流程）、WARNING（可恢复异常）、ERROR（严重错误）
- [ ] 网络异常时自动重试（最多 3 次）
- [ ] 页面结构变化时记录详细错误信息
- [ ] 日志输出到文件同时也在控制台显示

---

### US_06: 数据导出

**作为** 数据分析师  
**我希望** 将抓取的数据导出为 CSV 格式  
**以便** 在 Excel 或其他工具中进行分析

**验收标准 (DoD)**：
- [ ] 支持导出指定关键词的抓取数据
- [ ] CSV 包含所有必填字段
- [ ] CSV 文件命名包含日期和关键词（如 `products_拖把_2026-04-01.csv`）
- [ ] 导出时自动处理特殊字符（避免 CSV 格式错误）

## 🏗 架构决策 (ADR)

| 编号 | 决策点 | 最终选择 | 理由与权衡 |
| :--- | :--- | :--- | :--- |
| **ADR-01** | 爬虫框架选型 | **Python + Playwright** | 相比 Selenium 更轻量，Stealth 模式可绕过大部分反爬，支持异步并发 |
| **ADR-02** | 数据存储方案 | **SQLite + JSONL 双存储** | SQLite 便于查询去重，JSONL 便于数据交换和备份，兼顾两者优势 |
| **ADR-03** | 反爬策略 | **代理池 + 频率控制 + UA 轮换** | 多层防护，免费代理先测试，成功率<70% 转商业服务，控制成本 |
| **ADR-04** | 交互方式 | **命令行 MVP → Web 界面二期** | 快速验证核心价值，避免前期过度投入 UI，二期由少平负责前端 |
| **ADR-05** | 数据更新策略 | **保留历史 + is_latest 标记** | 支持价格/销量趋势分析，同时方便查询最新数据 |
| **ADR-06** | 开发语言 | **Python 3.11** | 爬虫生态成熟，Playwright 支持好，团队熟悉 |

## 📊 数据字段定义

### 必填字段

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| product_id | string | 商品唯一 ID | "675345678901" |
| title | string | 商品标题 | "家用拖把自动挤水免手洗" |
| price_min | number | 最低价格（元） | 15.8 |
| price_max | number | 最高价格（元） | 28.5 |
| sales | number | 销量 | 12580 |
| supplier_name | string | 供应商名称 | "义乌市 XX 家居用品厂" |
| supplier_level | string | 供应商等级 | "实力商家" |
| product_url | string | 商品详情页 URL | "https://detail.1688.com/xxx.html" |
| main_image_url | string | 主图 URL | "https://cbu01.alicdn.com/xxx.jpg" |
| thumbnail_urls | array[string] | 缩略图 URL 列表 | ["url1", "url2", ...] |
| collected_at | datetime | 采集时间戳 | "2026-04-01T12:00:00Z" |

### 选填字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| review_count | number | 评价数量 |
| repurchase_rate | string | 复购率 |
| ship_from | string | 发货地 |
| min_order_qty | number | 最小起订量 |
| tags | array[string] | 商品标签/属性 |

### 系统字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| is_latest | boolean | 是否为最新版本 |
| created_at | datetime | 记录创建时间 |
| updated_at | datetime | 记录更新时间 |
